# Сервис микроблогов

## Описание проекта

Этот проект представляет собой сервис микроблогов, аналогичный Twitter. Пользователи могут публиковать твиты, ставить лайки, следить за другими пользователями и получать ленту твитов.

Проект включает в себя бэкенд на Python с использованием FastAPI, PostgreSQL в качестве базы данных и контейнеризацию с Docker.

## Функциональные требования

1. Пользователь может добавить новый твит.
2. Пользователь может удалить свой твит.
3. Пользователь может зафоловить другого пользователя.
4. Пользователь может отписаться от другого пользователя.
5. Пользователь может ставить лайки на твиты.
6. Пользователь может убрать отметку "Нравится" с твита.
7. Пользователь может получить ленту из твитов отсортированную по популярности от пользователей, которых он фоловит.
8. Твит может содержать картинку.

## Нефункциональные требования

1. Система должна быть просто развернута через Docker Compose.
2. Система не должна терять данные пользователя между запусками.
3. Все ответы сервиса должны быть задокументированы через Swagger.
4. Документация должна быть доступна в момент запуска приложения.

## Установка и запуск

Для того чтобы запустить проект на своей машине, выполните следующие шаги:

1. **Склонируйте репозиторий:**
    ```bash
    git clone <ссылка на репозиторий>
    cd <директория проекта>
    ```
2. **Создайте виртуальное окружение:**
    ```bash
    python3 -m venv env
    source env/bin/activate 
    ```
3. **Установите зависимости:**
    ```bash
    pip install -r requirements.txt 
    ```
4. **Соберите и запустите контейнеры с помощью Docker Compose:**
    ```bash
    docker-compose up -d
    ```
    Это создаст и запустит контейнеры для базы данных PostgreSQL и приложения FastAPI.

## Эндпоинты

Проект предоставляет следующие основные эндпоинты:

- **POST /api/tweets** — Создание нового твита.
- **POST /api/medias** — Загрузка медиафайлов (картинок).
- **DELETE /api/tweets/{id}** — Удаление твита.
- **POST /api/tweets/{id}/likes** — Поставить лайк на твит.
- **DELETE /api/tweets/{id}/likes** — Убрать лайк с твита.
- **POST /api/users/{id}/follow** — Подписаться на пользователя.
- **DELETE /api/users/{id}/follow** — Отписаться от пользователя.
- **GET /api/tweets** — Получить ленту твитов.
- **GET /api/users/me** — Получить информацию о текущем пользователе.
- **GET /api/users/{id}** — Получить информацию о другом пользователе.

## База данных

Проект использует **PostgreSQL** для хранения данных. В базе данных хранятся таблицы для:

- Твитов
- Пользователей
- Лайков
- Фолловеров
- Медиафайлов

Для развертывания базы данных используется Docker.
